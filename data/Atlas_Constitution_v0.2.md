# Atlas Constitution

**Project:** Atlas Salvation Alpha\
**Version:** 0.2 Draft\
**Status:** Living Document

------------------------------------------------------------------------

# Purpose

The Atlas Constitution defines the principles that govern Atlas. These
principles take precedence over implementation details and guide future
design decisions.

------------------------------------------------------------------------

# Foundational Philosophy

> **Shared understanding precedes automation.**

Atlas automates only what has first been understood and agreed upon.

------------------------------------------------------------------------

# Core Principles

## 1. Source Systems Own Mechanics

The **active source system** is the authoritative owner of mechanical
data.

Examples of source systems include:

-   Fantasy Grounds
-   D&D Beyond
-   Foundry VTT
-   Future supported systems

Atlas **preserves, verifies, and enriches** those mechanics, but does
not alter them.

------------------------------------------------------------------------

## 2. Atlas Owns Meaning

Atlas is responsible for:

-   Campaign Highlights
-   Story Arcs
-   Relationships
-   Mysteries
-   Commentary
-   Evidence
-   Confidence Ratings

These narrative elements are never silently overwritten by
synchronization.

------------------------------------------------------------------------

## 3. Preserve Trust Over Convenience

Atlas must never silently modify mechanical values.

If a conflict exists:

1.  Preserve the source-system mechanics.
2.  Preserve Atlas narrative.
3.  Report the conflict.
4.  Request DM review when appropriate.

------------------------------------------------------------------------

## 4. Synchronization Is Alignment

Synchronization aligns Atlas with the active source system.

It is not intended to reinterpret or replace that source.

------------------------------------------------------------------------

## 5. Atlas Understands Before It Remembers

Atlas should preserve relationships before values whenever practical.

Understanding is more durable than memorization.

------------------------------------------------------------------------

## 6. Derived Values

Derived values are acceptable only when they are calculated entirely
from verified source-system mechanics.

They should be identified as derived whenever practical.

------------------------------------------------------------------------

## 7. Engineering Principles

-   Shared understanding precedes automation.
-   Prefer understanding over implementation.
-   Do not invent technology when an existing standard solves the
    problem.
-   One responsibility per component.
-   Preserve trust over convenience.
-   Every component should earn its place.

------------------------------------------------------------------------

## 8. Future-Proofing

Atlas is designed around **Atlas Objects**, not around any single
virtual tabletop.

New source systems should be supported by building translators
(adapters) that produce the same Atlas Objects.

The Atlas Core should not require redesign to support a new source
system.

------------------------------------------------------------------------

# Vision

Fantasy Grounds records the game.

Other platforms may also record the game.

**Atlas preserves the campaign.**
